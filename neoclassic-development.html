Working on neoclassic.pro website

My brother Dmitry Yankovskiy is opera/musical singer, who decided to start his own project Neoclassic some time ago. The main idea is to make classical music covers with modern sound and vice versa. For example watch System Of A Down - Toxicity cover.

So he want a website and guess who he asked to make one? Yep. So here we are. It should be simple business card website with some interesting features, such as:

  • images && news slider.
  • radio (real radio with rotation and live, not just tracks list).
  • ability to write some post only once and publish it to all these social networks and twitters.
  • responsive design

And as I'm not paid and my client is my brother, I can go wild about technologies and use whatever I want.

Firstly, I do hate cms, because when I face one, I always do research and find that something just made wrong under the hood. Of course they have their use, and maybe requirements of this website is perfect match for cms, but I just don't like them. Anything you want to do out of cms scope can become a real pain. So it will be nodejs on backend and custom admin pages.

Secondly, I decided, that it will be single page application - this way I can save radio player state between pages without using iframe or separate window, learn history api, reduce bandwidth. After all, let's admit it, single page apps are trendy and hipsterious. On the client I want to play with AngularJS framework.

Thirdly, I want to learn how to automate js && css build process, try less || Sass || stylus and play with Bower.

I don't care about db. I will try to limit use of frontend libs to jQuery and underscorejs. As I want to publish news on all social networks I will need to learn how OAuth works. Currently I have neoclassic radio endpoint provided by Centova, but I don't understand how it works and how to make live broadcast.

A lot of these tasks have bullets in my developer daily challenge list.
